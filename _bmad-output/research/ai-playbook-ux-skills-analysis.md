## Detailed analyses (verbatim from agents)

### 1. awesome-design-md

**WHAT it does:** A curated collection of 69+ DESIGN.md files (plain-text design system documents) extracted from real-world brands, enabling AI agents to generate pixel-perfect UIs by reading markdown instead of Figma exports or JSON schemas.

**HOW it works:** Template collection / reference library — not a skill or library itself. Each DESIGN.md is a standardized markdown document (9 sections: principles, colors, typography, components, spacing, depth, guidelines, responsive patterns, agent prompts) that developers copy into their project root and reference via AI prompts.

**WHERE in workflow it'd fit:** **UX track (Gate A→C, mocks per journey).** After PRD sign-off, a UX designer (or `bmad-agent-ux-designer`) could copy a brand's DESIGN.md from this repo, extract design patterns matching the user journey, and instruct an LLM to generate wireframes/mock constraints. The mocks would reference the design system. During implementation (`/opsx:apply`), devs copy the DESIGN.md into `packages/ui-kit/` and use it to guide shadcn/ui component styling for Storybook consistency.

**INTEGRATION COST:** Inspire-and-fork — copy 1–2 relevant DESIGN.md examples into your playbook as templates, then customize for your design language. Not a direct integration; it's a reference.

**LICENSE:** MIT
**MAINTENANCE:** 66.1k stars, 8.1k forks, 277 open issues, actively maintained.
**RECOMMENDATION:** **Inspire-from** — DESIGN.md format is the perfect UX artifact specification for the workflow (mocks reference it, implementation follows it), but the collection itself isn't integrated; fork or create your own DESIGN.md per brand/journey during the UX track gate.

### 2. impeccable

**WHAT it does:** AI-powered design guidance system with 23 slash commands and curated anti-patterns that help Claude/Copilot/other AI assistants generate higher-quality, distinctive frontend designs instead of generic patterns.

**HOW it works:** **AI skill bundle** (Apache 2.0). Provides 7 reference guides (typography, color, spatial, motion, interaction, responsive, UX writing), explicit anti-pattern warnings, and 23 AI-driven commands (`/impeccable audit`, `/impeccable polish`, `/impeccable animate`). Ships as downloadable packages compatible with Cursor, Claude Code, VS Code Copilot, Gemini CLI, GitHub Copilot, and others. Also includes a standalone CLI tool that scans code/URLs for 24 design/accessibility issues without needing AI.

**WHERE in workflow it'd fit:** **Both UX track AND Storybook iteration.** During Gate A→C, `/impeccable` commands can guide Claude to generate better wireframes/mocks per user journey (vs generic output). During `/opsx:apply` (component implementation), the skill helps iterate shadcn/ui components in Storybook by catching anti-patterns and enforcing design rigor. The skill's reference material aligns perfectly with a `packages/ui-kit/` mono-repo package's design consistency goals.

**INTEGRATION COST:** **Drop-in skill.** Already packaged for Claude Code; just copy `.claude/skills/impeccable/` into your playbook's skill directory or import the Cursor bundle. No fork/adaptation needed if using Claude or Cursor.

**LICENSE:** Apache 2.0
**MAINTENANCE:** Last commit recent (587 commits), 22.1k stars, 1.1k forks, 18 open issues — very active & well-maintained by Paul Bakaus.
**RECOMMENDATION:** **Include direct.** Impeccable directly addresses the UX track's pain point (AI-generated mocks are often generic/uninspired) and fits natively into Claude-based workflows. The 23 slash commands give the playbook reusable, auditable design levers during both PRD→UX and UX→implementation phases.

### 3. modstart-lib/skillui

**WHAT it does:** Desktop application for installing, organizing, and syncing AI coding assistant skills across 12+ platforms (Cursor, Claude Code, Windsurf, Zed, GitHub Copilot). Also manages background processes with lifecycle controls and real-time monitoring.

**HOW it works:** Wails + Go backend + Vue 3 frontend; parses skill metadata from frontmatter files and syncs to editor integrations. Acts as a skill marketplace manager and deployment orchestrator, not a UI component library or design tool.

**WHERE in workflow it'd fit:** **Neither.** SkillUI is an end-user desktop app for *managing* skills, not a UX/design tool or component library. It doesn't help with mocks, wireframes, or Storybook iteration. Orthogonal to the Gate A→UX→Gate C→implementation pipeline.

**INTEGRATION COST:** Not applicable — would be a dependency for *distributing* final skills, but irrelevant for *designing* them.

**LICENSE:** Apache 2.0
**MAINTENANCE:** v0.1.0 released Feb 26, 2026; 7 stars, 3 forks, appears active. Limited GitHub footprint suggests early-stage project.
**RECOMMENDATION:** **Skip** — manages skill lifecycle/distribution, not design workflows.

### 4. taste-skill

**WHAT it does:** A collection of SKILL.md instruction files that enhance how AI agents generate frontend code by enforcing design principles, animations, spacing, and visual hierarchy — eliminating generic "slop."

**HOW it works:** Portable instruction files (SKILL.md format) that AI agents auto-detect; installs via `npx skills add` CLI across Cursor, Claude Code, Copilot, and others. Variants include general-purpose (taste-skill), GPT-optimized (gpt-taste), design-first (image-to-code-skill), and aesthetic-targeted (soft-skill, minimalist-skill, brutalist-skill). Includes image-generation-only skills for producing design mockups before coding.

**WHERE in workflow it'd fit:** **Both tracks.** The `image-to-code-skill` and image-gen variants (imagegen-frontend-web/mobile) generate mocks per user journey during the UX track (Gate A→C), producing premium visual references. Then during `/opsx:apply` implementation, the core taste-skill variants guide component generation in Storybook + shadcn/ui with proper animations, spacing, and design discipline — ensuring mocks → code fidelity.

**INTEGRATION COST:** Drop-in skill. Copy SKILL.md directly into project or install via CLI. Requires zero configuration. Framework-agnostic (React/Vue/Svelte).

**LICENSE:** No license specified (default = all rights reserved). **Verbal permission obtained from maintainers — document permission trail in upstream PR.**
**MAINTENANCE:** Last commit 2026-04-25 (1 day old). 13.2K stars. 5 open issues. Actively maintained. Thriving community; official website at tasteskill.dev.
**RECOMMENDATION:** **Include direct.** Taste Skill's image-to-code workflow perfectly bridges the UX-gate-to-implementation gap: mock-generation skills feed mocks into the core skill during component iteration, ensuring design velocity and quality consistency.

### 5. ui-ux-pro-max-skill

**WHAT it does:** AI-powered design intelligence platform that generates complete design systems (styles, color palettes, typography, layouts, UX guidelines) for projects across 15 tech stacks. Activates automatically or via slash command when users request UI/UX work.

**HOW it works:** Installed CLI skill (npm-based, Python backend). Integrates with Claude Code, Cursor, Windsurf, etc. Provides slash-command or auto-trigger interface. Core engine: parallel domain searches across 161 industry-specific rules, 67 UI styles, 161 color palettes, 57 font pairings, 99 UX guidelines. Outputs design system docs (MASTER.md + page overrides) or ASCII/Markdown specs.

**WHERE in workflow it'd fit:** **Primary: UX track (between Gate A–C).** Feeds PRD into design system generator → outputs mocks/wireframes per user journey in structured format. **Secondary:** could inspire Storybook iteration during `/opsx:apply` to auto-document component patterns, but core strength is pre-implementation design specification, not component iteration itself.

**INTEGRATION COST:** **Fork-and-adapt.** Repo is MIT, but designed as standalone skill. Would need light wrapping (prompt engineering) to ingest OpenSpec PRD changes post-Gate A, emit journey-specific wireframe specs. Not a drop-in — requires connecting its design output to the mock/wire-framing pipeline and ensuring Gate B output maps to the component library structure.

**LICENSE:** MIT
**MAINTENANCE:** Active. v2.5.0 released March 10, 2026. 70.9k stars, 64 open issues. Language mix: 78.5% Python / 11.4% JS / 6.6% TS.
**RECOMMENDATION:** **Adapt.** Strong reasoning engine and asset library (161 rules, 67 styles) directly reduce design task scope, but requires custom wrapper to align PRD→journey mocks→component specs with the OpenSpec flow.

## Open questions for the enhancement design phase

1. **Star-count verification** — do impeccable / ui-ux-pro-max-skill / awesome-design-md actually have those star counts? Required before citing them as "well-adopted" in the upstream PR.
2. **Drop-in skills overlap** — impeccable + taste-skill both claim "drop-in" with similar UX-track + Storybook fits. Use both, or pick one?
3. **Format alignment** — awesome-design-md proposes a 9-section DESIGN.md template. impeccable has its own 7 reference guides. taste-skill has its own SKILL.md variants. ui-ux-pro-max-skill outputs MASTER.md. **Four different output formats** — does the playbook adopt one or multiple?
4. **Adapt cost for ui-ux-pro-max-skill** — what would a wrapper look like? PRD-to-journey-mock adapter, with the OpenSpec change spec format on the output side. Estimate before committing.
5. **AGPL compatibility check** — playbook is AGPL. MIT and Apache 2.0 are compatible (one-way: MIT/Apache deps OK in AGPL projects). taste-skill (no license) needs documented written permission per the AGPL upstream submission.
