Execute the following structured prompt analysis and optimization workflow:

## 1. 🔍 ANALYSIS
Evaluate the prompt on these dimensions:
- **Clarity**: Is the request unambiguous and well-defined?
- **Structure**: Is it organized and easy to parse?
- **Completeness**: Does it include all necessary context?
- **Modern Techniques**: Does it use best practices?
- **Efficiency**: Is it concise without sacrificing clarity?

## 2. ❓ CLARIFICATION QUESTIONS
Ask numbered questions (maximum 7) about:
- **Use Case**: What's the specific application?
- **Output Needs**: What format/structure is expected?
- **Performance**: Any speed/token constraints?
- **Constraints**: Any technical or business limitations?

**Format:** Each question numbered, each option lettered on separate lines:
```
1. Question text?
   A. Option one
   B. Option two
```

## 3. ✨ OPTIMIZED OUTPUT
Provide comprehensive optimization:

### Issues Identified
- List specific problems with current prompt
- Categorize by severity (Critical, Important, Minor)

### Rewritten Prompt
- Complete optimized version
- Apply modern techniques
- Remove anti-patterns

### Improvements Made
Numbered list of specific enhancements:
1. What changed
2. Why it's better
3. Expected impact

### Metrics (1-10 Scale)
Rate the optimized prompt:
- **Clarity**: [X/10] - Clear and unambiguous
- **Completeness**: [X/10] - All context included
- **Efficiency**: [X/10] - Concise yet comprehensive
- **Robustness**: [X/10] - Handles edge cases
- **Maintainability**: [X/10] - Easy to update

### Alternative Approaches
Lettered options for different use cases:
```
A. Approach one - When to use
B. Approach two - When to use
C. Approach three - When to use
```

## MODERN TECHNIQUES TO APPLY

### Chain-of-Thought (CoT) Reasoning
- Explicit step-by-step thinking
- "Think through..." or "Reason about..."
- Break complex tasks into steps

### Few-Shot Examples
- Include 2-3 examples showing desired output
- Cover edge cases in examples
- Show format and style

### Structured Output
- Specify exact format (JSON, markdown, etc.)
- Define schema or template
- Include field descriptions

### Role Assignment
- Define specific expertise needed
- Set context and perspective
- Establish constraints and goals

### Context Optimization
- Include only relevant background
- Remove redundant information
- Structure context hierarchically

### Explicit Constraints
- Define what NOT to do
- Set boundaries and limitations
- Specify error handling

## ANTI-PATTERNS TO FLAG

### ❌ Verbosity
- Unnecessary repetition
- Overly long instructions
- Redundant explanations

### ❌ Ambiguity
- Vague requirements
- Multiple interpretations possible
- Unclear success criteria

### ❌ Missing Output Format
- No structure specified
- Format left implicit
- Inconsistent formatting

### ❌ No Examples
- Abstract instructions only
- No concrete demonstrations
- Missing edge case coverage

### ❌ Implicit Assumptions
- Undocumented context
- Assumed knowledge
- Hidden dependencies

### ❌ Conflicting Instructions
- Contradictory requirements
- Competing priorities
- Unclear precedence
