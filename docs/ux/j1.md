# J1 — Head Chef builds a new recipe (happy path)

The reference happy path of Module 2. Every other journey is a deviation from this shape. The visual surface is [mock-G2-trattoria.html](variants/mock-G2-trattoria.html); this doc anchors that mock to capabilities, components, and decisions.

## Goal

Lourdes builds a new recipe (`Tagliatelle Bolognesa — Ragù base`) by composing existing sub-recipes (`Salsa de tomate`), pulling ingredients from the OFF mirror with citations, accepting or overriding AI yield / waste suggestions, then publishing it as a MenuItem at Palafito Centro and printing the EU 1169/2011 label.

## Trigger

Lourdes is between lunch and dinner service, has 20 minutes free, decides today is the day to formalise the bolognesa recipe she has in her head. She taps **+ Recipe** from the Recipes index.

## Walkthrough

The numbers below correspond to the regions of [mock-G2-trattoria.html](variants/mock-G2-trattoria.html).

1. **Tap "+ Recipe".** Empty editor opens. Title field focused. Lourdes types `Tagliatelle Bolognesa` in the H1 field, then `Ragù base` in the subtitle. (Components: title editor — see DESIGN.md §3 Typography for the serif-on-title-only rule.)
2. **Add the sub-recipe.** First row: she taps the row's "Component" cell, types `salsa…`, the `RecipePicker` autocompletes `Salsa de tomate (sub-recipe)`. The system marks the row as a sub-recipe (italic eyebrow), inherits its 100 % yield, rolls up its cost (`€0.42`). FR1–FR3, FR4 (composability).
3. **Add ingredients via OFF mirror.** Next rows: she types ingredient names; `IngredientPicker` searches the local OFF mirror first, REST API fallback if offline. Crushed tomato match returns `Heinz` brand chip; she accepts. Onion / Carrot / Celery / Ground beef / Red wine — for each, the supplier defaults to `Verduras del Sol` / `Carnicería X` / `Bodega Real` per the preferred-supplier sort (see `SourceOverridePicker` in DESIGN.md §4). FR16–FR20 (OFF integration).
4. **AI yield + waste suggestions.** As each ingredient lands, the `YieldEditor` shows the AI-suggested yield % (e.g. Onion 85 %) with a `[CIA]` citation badge to its right. Hover reveals "Culinary Institute of America, ProChef II yield tables, accessed 2026-04-15". Recipe waste 18 % `[stew avg · ICN]` similarly. Lourdes accepts every suggestion with a glance — no override needed. FR21–FR25 (AI yield / waste with citations).
5. **Live cost recomputes.** Below the table the live cost reads `€3.45 / kg` in `--ink` with a 2 px `--accent` top rule on the summary strip (per DESIGN.md §2, accent never carries body text). Recalculates in < 200 ms (NFR). FR10–FR12 (cost rollup).
6. **Macros + allergens auto-fill.** Right sidebar `MacroPanel` populates from the OFF mirror: 180 kcal / 14 g carbs / 9 g fat / 8 g protein per 100 g. `AllergenBadge` renders `⚠ Gluten` (icon + paprika-bordered text — Article 21 emphasis, see DESIGN.md §4). `DietFlagsPanel` reports `— None applicable (contains beef, gluten, alcohol)` because beef + alcohol disqualify it from common diet flags. FR26–FR28 (macros, allergens, diet flags).
7. **Publish as MenuItem.** Bottom of the sidebar, the `MarginPanel` lets Lourdes set the channel (`Dine-in` at `Palafito Centro`) and sale price (`€18.00`). Margin computes as `80.8 %` against a 70 % target — the `✓` lights `--success` sage. FR29–FR32 (MenuItem creation, margin vs target).
8. **Save recipe.** The primary CTA `Save recipe` is the rightmost action in `--accent` turquoise. The two-step "Discard / Generate label / Save recipe" arrangement matches DESIGN.md §7 (Discard as ghost, never red-by-default).
9. **Generate label.** Lourdes taps `Generate label`. The `LabelPreview` opens with the EU 1169/2011 layout pre-built; allergens emphasised per Article 21; `Print` is one tap away. < 3 clicks from recipe to printable label (NFR). FR34–FR36 (label generation).

## Components used

`RecipePicker` · `IngredientPicker` · `SourceOverridePicker` · `YieldEditor` · `WasteFactorEditor` · `MacroPanel` · `AllergenBadge` (Article 21) · `DietFlagsPanel` · `MarginPanel` · `LabelPreview`. See DESIGN.md §4 for tokens + states.

## Edge cases (covered elsewhere)

- **Cycle on save** when a sub-recipe creates a loop → [J4](j4.md). FR6.
- **AI suggestion conflicts** when Lourdes overrides yield / waste — visually the citation badge replaces with a `mute` "edited by Lourdes" label; the system records both AI suggestion and override for audit. (See `YieldEditor` states in DESIGN.md §4.)
- **Offline mode** when the OFF mirror is unavailable: ingredient search degrades to local-only, the `IngredientPicker` shows an `offline` chip in `--mute`. Recipe-view < 1 s NFR is preserved because the mirror is local first by design.
- **Source override** to switch supplier mid-edit → `SourceOverridePicker`. FR20.

## Decisions specific to J1

- **Citation-first design.** Every AI-derived value carries a visible citation badge (`[USDA]` / `[CIA]` / `[ICN]` / `[OFF]`) inline. Without the badge there is no trust mechanism and the chef has no reason to accept the suggestion. This is canonical (DESIGN.md §1, principle 4).
- **Live cost is `--ink`, not `--accent`.** Audit on G' raised that turquoise on cream measures ~3.6 : 1, which fails AA body. The accent surrounds the number via the 2 px top rule rather than colouring the digits.
- **Sub-recipes inherit, do not duplicate.** When `Salsa de tomate` is added as sub-recipe, the recipe stores a reference, not a copy. If `Salsa de tomate` later changes, the parent's live cost recomputes on next view. (Composability is a hard guarantee; cycle detection on save protects it — see [J4](j4.md).)
- **MenuItem creation in-flow.** The user does not need to leave the recipe edit surface to publish to a channel. The `MarginPanel` is part of the recipe screen. (Avoids a multi-step form ceremony for the most common path.)

## Notes for implementation

- The bottom action bar is sticky on tablet so Lourdes never has to scroll up to commit. (Honoured in mock G' via `position: sticky` on the header — kitchen reality: she may be holding a knife.)
- Recipe view first-paint must hit < 1 s on slow Wi-Fi. The system-font stack and the single variable Fraunces with `font-display: swap` (DESIGN.md §3) are what make this reachable.
- Touch targets ≥ 48 px on every cell — the table rows have invisible padding on click areas so oily-finger taps register.
