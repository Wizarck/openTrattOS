# J8 — Owner reads the AI observability dashboard (operations path)

The cost & reliability review surface for everything AI-mediated in nexandro: vision-LLM extractions, agent-driven MCP calls, embedding generation. Iker (Owner, all-sites) and Marta (Manager, per-location) read this surface to answer three recurring questions: *Cuánto estamos gastando · en qué · y vamos bien o mal*. The visual surface is [mock-j8-ai-observability.html](variants/mock-j8-ai-observability.html); this doc anchors that mock to the 6-core + 2-optional widget hierarchy.

## Goal

In a 2-second glance Iker sees (i) the current error rate (semaphore), (ii) the total monthly cost vs. budget, and (iii) the burn-rate runway. In a 10-second drill he answers "qué capacidad / qué modelo / qué tag dispara el coste". In a 30-second drill he reads the top 5 failures and tags them for action.

## Trigger

Monday morning, weekly review. Iker opens `/ai/observability` from the top nav. Alternatively, the dashboard auto-opens when a budget-tier alert fires (`info=50% / warn=75% / error=90% / fatal=100%` per NFR-OBS-10) — the email notification deep-links here.

## Walkthrough

The numbers below correspond to the widget grid of [mock-j8-ai-observability.html](variants/mock-j8-ai-observability.html). The hierarchy is Caravaggio-style: the alarm/status row reads first, the spend row second, the usage shape third, the failure list fourth.

1. **Status row (top, 3 widgets, eyebrow + headline pattern).**
   - **`ErrorRateWidget`** (left). Semaphore dot + 24-hour mini-timeseries. Green dot `--success` = error rate < 1 %; amber = 1-5 %; red `--destructive` = > 5 %. Headline: `0.4 % en las últimas 24h`. Below: a sparkline of the past 24 hours. (FR45, NFR-OBS-3.)
   - **`CostTotalWidget`** (centre). Current-month spend in `--ink`, large display (`€ 84,21`). Below in `--mute`: `de un presupuesto mensual de €120,00`. Below that: thin bar showing % consumed (70 %). The bar shows the tier-cross thresholds as 4 small ticks at 50 / 75 / 90 / 100 — bar fill reaches just past the warn tick. (FR46, NFR-OBS-10.)
   - **`BudgetStatusWidget`** (right). Tier badge `Warn · 70%` in amber. Below in `--mute`: `Quedan ~13 días al ritmo actual`. The burn-rate calc uses trailing-7-day average daily spend ÷ remaining budget (per architecture sub-decision). (NFR-OBS-10, eligia-dashboard cross-pollination.)

2. **Spend row (middle, 3 widgets, breakdown).**
   - **`CostByCapabilityWidget`** (left). Horizontal bar chart, 5 capabilities listed: `inventory.ingest-invoice-photo` (47%), `recall.generate-dossier` (23%), `haccp.record-ccp-reading` (12%), `inventory.ingest-product-photo` (10%), `recall.search-incident` (8%). Bars in `--accent`; sorted descending. Hover/tap reveals exact € and call count. (FR46, NFR-OBS-4.)
   - **`CostByModelWidget`** (centre). Same horizontal bar pattern, models listed: `gpt-oss-vision-72b` (54%), `gpt-4o-mini` (28%), `claude-3-5-sonnet` (12%), `gpt-oss-72b` (6%). Surfaces the dependency on each provider — directly answers the "what would happen if we drop one model" question. (NFR-OBS-4.)
   - **`CostByTagDrillDown`** (right, OPTIONAL widget #7 per architecture). Horizontal bar chart grouped by `nexandro.tag` JSONB attr. Bars: `recall-investigation` (31%), `friday-delivery-batch` (28%), `weekly-ccp-bulk` (19%), `photo-ingest-adhoc` (14%), `(no-tag)` (8%). Tags are caller-supplied free-form labels. Operator action: identify which workflow drives spend; tag taxonomy is the operator's, not the system's. (NFR-OBS-2 extended per eligia cross-pollination.)

3. **Usage shape row (3rd row, 1 wide widget).**
   - **`UsageByDayHeatmap`** (full width). 7-day × 24-hour heatmap. Rows = days (lunes → domingo); columns = hours (00 → 23). Cells coloured in OKLCH `--accent` with stepped lightness for usage intensity. Friday 09-12 reads darkest (delivery photo-ingest peak). Sunday all-day reads near-empty. The heatmap answers "when do we actually use AI?" — drives staff-allocation decisions. (NFR-OBS-5.)

4. **Failure row (bottom, 1 wide widget).**
   - **`Top5FailuresWidget`**. Flat list, 5 rows: each row = error type + count + last occurrence + `Ver eventos →` link. Rows: `1. Vision-LLM timeout · 14 últimas 24h · hace 2h`, `2. Pricing row not found · 7 últimas 24h · hace 5h`, `3. OTLP exporter 503 · 3 últimas 24h · hace 11h`, etc. Tap a row to drill into the audit_log filtered to those failures (links to [J12](j12.md)). (NFR-OBS-6, NFR-OBS-7.)

5. **Range selector (sticky header).** Above the widget grid, a chip group: `24h` (selected), `7d`, `30d`, `Este mes`, `Mes pasado`. Every widget re-renders client-side on chip change (LRU cache hit, no spinner — per architecture cache sub-decision). Default `24h` for the morning-review case; `Este mes` for the budget-review case.

6. **Plug-in hint (footer eyebrow).** A single mute line at the bottom of the dashboard: `OTLP exporter activo → langfuse.nexandro.local · ver configuración →`. Surfaces that the same OTel spans flowing into this dashboard also stream to whatever external tool the operator has configured (Langfuse / LangSmith / Phoenix / Datadog). No widget for the external tool — the link is enough. (Per [[feedback_agent_as_ux_surface]] — embedded dashboard is lightweight; external tool plug-in is the value.)

## Edge cases

- **First boot, no telemetry yet.** Every widget renders its empty state with a mute body line: `Sin datos en este rango.`. No skeletons, no spinners — the empty state is the data. The range chips still work; switching to `30d` may surface earlier data.
- **OTLP exporter offline.** A `--warn-bg` strip mounts above the dashboard: `OTLP exporter inactivo desde 02:14 CEST · los spans se persisten localmente y se reenviarán al recuperar el endpoint`. The dashboard continues to render from the local rollup table (ADR-030 + ADR-029 — local-first design).
- **Budget tier crossed mid-render.** The `BudgetStatusWidget` re-renders with the new tier badge + colour. An audit_log envelope is written (`ai_budget_tier_crossed`); no modal interrupt — Iker reads the dashboard, not a popover.
- **Tag drill-down with zero tagged calls.** Widget #7 hides itself with a mute footer line: `Sin tags · activa el atributo nexandro.tag al emitir spans →` linking to docs. Surfaces should NOT show empty bars (visual noise).
- **Single capability dominates > 80 %.** The capability list shows just one full bar + a mute footer `Solo una capacidad activa` — no need to render 4 near-empty bars.
- **Cache miss + Postgres slow.** Per ADR-030 dashboard cache sub-decision: the LRU returns last-known-good payload with `X-Nexandro-Stale: true`. Frontend reads the header and renders a `--mute` eyebrow strip at the top: `Datos en caché · refrescando…`. Removed when fresh payload lands.
- **Operator is a Manager, not Owner.** Same dashboard, scoped to their assigned locations only. The header eyebrow reads `Tu alcance: 1 de 3 locales` so they don't mistake their view for the all-org total. Tag drill-down stays per-org (tags are operator-supplied; scoping them per-location would over-fragment).

## Decisions specific to J8

- **Three rows × Caravaggio hierarchy: status → spend → shape → failures.** The Owner's recurring scan starts with "is anything red?" (status row), continues with "where is the money?" (spend row), then "when do we use it?" (shape row), then "what's broken?" (failures row). This is the order the eye should land. Putting failures at the top would burn the most expensive widget on a question that's usually-no; putting status at the bottom would force the operator to scroll for the most important signal.
- **Budget tier system is a widget, not a banner.** Eligia-dashboard puts the budget bar inline at the top of every dashboard view; the same pattern transfers. Banner alerts on budget arrive via email (per ADR-030 `BudgetAlertDispatcher`) — the dashboard widget is the in-product view, not the alert mechanism.
- **Soft-enforce budget, never block.** Per ADR-030 + eligia cross-pollination: even at 100 % fatal tier, AI calls keep going through. The badge turns paprika, the email fires, the audit_log records — but the operator decides cutoff, not the system. (Inversion of typical SaaS billing dashboards where over-budget = service stops; aligns with [[feedback_commoditizer_not_disruptor]] — we are not the gatekeeper.)
- **Tag drill-down is optional.** Many operators won't supply `nexandro.tag` on their spans initially. Widget #7 self-hides on zero data (vs. showing an empty grey box). When tags appear, the widget appears — discoverable by the data, not by toggles.
- **No per-call drill-down on this dashboard.** Tapping a model or capability bar does NOT drop into "show me every call". The per-call view is the audit_log browse ([J12](j12.md)) filtered to AI capabilities. Keeping that drill-down in the audit-log surface (where every other forensic question lives) avoids duplicating a forensic view inside an observability view.
- **OTLP plug-in hint is a single mute line.** The dashboard is read-side; the OTLP config is settings-side (`/settings/observability`). A widget for "external tools" would be marketing chrome — the line at the footer is enough. (DESIGN.md §1 principle 1 — restraint as discipline.)
- **Range default = 24h.** Monday-morning case wants "what happened this weekend?". The budget case wants "this month" — but the chip switch is one tap. Defaults serve the recurring case.
- **`--accent` for the bars; `--destructive` only on the error semaphore.** Spend is data, not danger. Painting cost bars red would conflate "expensive" with "wrong" — a Vision-LLM call worth €0.40 on a recall investigation is both expensive AND correct. Reserve `--destructive` for the actual error signal. (DESIGN.md §2 semantic colour.)

## Notes for implementation

- The dashboard reads from `ai_usage_rollup` via `dashboard.service.ts` per ADR-030. The LRU cache + stale fallback is in the service; the frontend just reads + reacts to the `X-Nexandro-Stale` header.
- The 24-hour timeseries on `ErrorRateWidget` is computed on the fly from rollup rows; rendered as a 60-point sparkline. Granularity per range: 24h = hour-resolution; 7d = hour-resolution; 30d = day-resolution; Este mes = day-resolution; Mes pasado = day-resolution.
- The heatmap reads `day_of_week × hour_of_day` directly from the rollup table — that's the partition the rollup is computed against. One query, one cache key.
- Widget #7 (`CostByTagDrillDown`) requires the span enricher to attach `nexandro.tag` (per ADR-030 main decision). Backwards-compat: spans without the attribute go to `(no-tag)` bucket; widget self-hides if the bucket is the only one.
- The plug-in hint at the footer is a static link if `NEXANDRO_OTEL_EXPORTER_ENDPOINT` env is set; otherwise it reads `Configura un endpoint OTLP en /settings/observability para conectar Langfuse / LangSmith / Datadog`.
- First-paint < 1 s on laptop. The widget grid is server-rendered for the chip-default range; chip changes are client-side cache reads.
- Accessibility: every widget has a `<h3>` semantic heading; bars and heatmap cells carry `aria-label` with full disclosure (`Inventory.ingest-invoice-photo: €39,58, 47% del total mensual`). The semaphore dot uses both colour AND a glyph (✓ / ⚠ / ✗) — never colour-only (DESIGN.md §1 principle 8 reinforced).
- Print stylesheet: the dashboard prints to a single landscape page; widgets reflow into a 2-column grid; sparkline + heatmap remain readable in greyscale (no colour-only encoding).
