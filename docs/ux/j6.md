# J6 — Multi-Location Owner runs a 2 AM recall investigation (crisis path)

The crisis spine of M3. Iker arrives on this surface because the phone has rung — a customer has been hospitalised, a regulator may call within hours, the EU 178/2002 four-hour clock is already counting. Every choice on the screen assumes he is in cortisol-tunnel mode: no decoration, no marketing, no onboarding hints, no choices that are not load-bearing. The visual surface is [mock-j6-recall-investigate.html](variants/mock-j6-recall-investigate.html); this doc anchors that mock to capabilities, components, and decisions.

## Goal

In ≤10 clicks and ≤8 minutes, Iker (i) finds the candidate lot(s) implicated in the incident, (ii) sees the consumption graph (which menu items, which locations, which timeframes), (iii) 86-flags affected menu items across every location, and (iv) dispatches the pre-formatted incident dossier to insurer + sanitary authority — without ever leaving this surface.

## Trigger

02:00. Iker's phone rings. A customer at Casa Aitona Algorta has been hospitalised; preliminary diagnosis suggests anisakis from raw fish. He opens nexandro on his phone. The home screen — when an open incident is unresolved or when the operator lands directly on `/recall/investigate` — collapses to a single visible CTA and a single search field. There is no top nav, no widget grid, no greeting. The interface assumes he is reacting, not deliberating.

## Walkthrough

The numbers below correspond to the regions of [mock-j6-recall-investigate.html](variants/mock-j6-recall-investigate.html).

1. **Crisis banner.** Top edge of the screen: a 4 px `--destructive` rule (paprika, never red-by-default — DESIGN.md §2 semantic colour). Centred eyebrow: `Investigación de incidente · 02:14 CEST · ventana legal 04:00`. The four-hour EU 178/2002 clock is rendered live, counting down in `tabular-nums` so digits do not dance as seconds tick. No icons, no logo, no theming flourish.
2. **Single search field.** One field, full width, autofocus on mount. Placeholder: `pescado crudo Algorta esta semana`. The field accepts free-form text and matches across lot code, supplier name, ingredient name, received-date phrasing, symptom keyword, and aggregate type (FR14). Touch target ≥ 56 px (above the standard 48 px because the operator is shaking).
3. **Candidate lot list.** Beneath the search field, the system surfaces up to 8 candidate lots ranked by recency (newer first), then by symptom-match score. Each row carries: lot code (Fraunces serif, the one wink), supplier name, received date in `el martes 09:30` natural-language form (not ISO), and an inline mute eyebrow with the menu items this lot has touched. Tapping a row drills into the forward trace. The list is the table — not a card grid, not a tile gallery (DESIGN.md §6 — flat lists for high-stakes scans).
4. **Forward-trace tree** (`RecallTraceTree`). On tap of a candidate lot, the search field collapses to a breadcrumb and the trace tree mounts in its place. The root node is the lot. Children: recipes that consumed this lot (with consumed-quantity badge). Grandchildren: menu items that compose those recipes. Great-grandchildren: locations × service-window leaves (`Algorta · miércoles cena`, `Bilbao · jueves comida`). The tree is rendered with a single accent rule down the left margin per level — no card containers, no nested boxes. Touch targets ≥ 48 px. SQL recursive CTE depth capped at 10 per ADR-028; if cap is hit, the leaf shows a mute `…profundidad excedida` eyebrow with a link to the partial dossier (an extreme edge case; should not happen in MVP scale).
5. **Sticky single-CTA bar.** Bottom of the screen, sticky: one button. Width = 100 % of the viewport minus the 24 px gutter. Label: `Detener servicio + Generar dossier`. Background `--destructive`, text `--accent-fg`. No secondary actions, no cancel button, no "save draft". The crisis surface has exactly one forward action; back-out is via the breadcrumb. The button height is 64 px (well above the 48 px minimum) because the operator's finger is not steady.
6. **Confirmation interstitial (one-tap, no modal stack).** Tap of the primary CTA opens a single confirmation strip (NOT a modal — DESIGN.md §6 "modal-as-first-thought" anti-pattern) directly below the button: `¿Cortar servicio en 3 locales + enviar dossier a 2 destinatarios?` with a single confirm pill `Sí, despachar ahora` and a ghost `Volver`. Confirm dispatches the 86-flag to all kitchen agent surfaces (FR17), generates the dossier PDF (FR18), and emails it to the pre-configured insurer + sanitary-authority contact lists (FR19). The strip then morphs in place to a receipt: `Dossier dispatched · 02:21 CEST · ver dossier →`. The link routes to [J7](j7.md).
7. **No empty state ceremony.** If the search returns zero candidates, the list region reads, in `--mute` body weight: `Sin coincidencias. Refina la búsqueda — o reporta sin lote conocido`. Below it, a single ghost link `Reportar sin lote conocido` opens a manual incident drafting flow (out-of-scope for J6; reaches the same dossier generator without a lot anchor).
8. **No theme switcher, no nav, no avatar.** The header is the eyebrow + clock. The footer is the sticky CTA. Everything between is the search → list → trace flow. The operator never has to decide where to look.

## Edge cases

- **Operator cannot type complete phrase.** Search runs on every keystroke after 200 ms debounce. The list re-ranks live. The field never blocks while results compute (DESIGN.md §1 principle 8 — the recurring case is the panicked operator with three letters typed).
- **Multiple candidate lots from the same supplier match.** All surface; the row eyebrow disambiguates by received-date + lot tail digits. The operator picks the one that matches the date the customer reportedly dined.
- **Trace depth excessive (sub-recipe → sub-sub-recipe → menu item).** The recursive CTE caps at depth 10 (ADR-028). If hit, the leaf carries `…profundidad excedida` mute eyebrow + a link to the partial-trace dossier. The dossier still dispatches; the partial-trace status is captured in the dossier metadata so the regulator sees the boundary.
- **Email dispatch fails (transient SMTP 5xx).** The 86-flag is already dispatched and persisted. The receipt strip reads `Dossier generado · envío en cola · 1 / 3 intentos` and 3-retry exponential backoff runs per NFR-REL-2. Final failure surfaces to Iker via dashboard alert. The dossier itself stays accessible via the receipt link — Iker can forward it manually as a fallback.
- **Insurer contact list not configured.** The dossier still generates and emails to the sanitary-authority contact only (if configured). A mute eyebrow above the receipt reads `Aseguradora no configurada — añadir en /settings/recall-contacts`. Never block dispatch on incomplete configuration; the regulator deadline does not wait.
- **Zero lots found.** Fallback to `Reportar sin lote conocido` flow (out-of-scope for J6 mock; same dossier generator without lot anchor).
- **Operator dismisses confirmation.** The `Volver` ghost returns to the trace tree without dispatching. State preserved; the search field still holds the query.
- **Multiple incidents open simultaneously.** Each lives at `/recall/investigate/:incidentId`; the home `/recall/investigate` opens the most recent. Surface design unchanged — this is an indexing concern, not a UX concern.

## Decisions specific to J6

- **Crisis-mode is the entire surface, not a banner overlay.** The naive instinct is "show the normal app + add a red banner". That fails the persona: at 02:00, the operator's eye is hunting for the action, and chrome around it is noise. The whole page is the crisis surface; the rest of nexandro is reachable only via the breadcrumb after a result is selected. (DESIGN.md §1 principle 1 — restraint as discipline.)
- **Single sticky CTA, not a two-button row.** A "Cancel" or "Save draft" alongside the primary action is a fork the operator should not face. Back-out is via the breadcrumb (a passive affordance, not a competing button). The single button removes the decision; the confirmation strip restores the safety net.
- **Confirmation strip, not a modal.** Modals stack the decision on top of the visual context the operator just built (the trace tree). A strip beneath the button keeps the context visible and grounds the confirmation in the work the operator just did. (DESIGN.md §6 modal-as-first-thought anti-pattern.)
- **Live four-hour countdown is rendered, not a static badge.** The clock is the load-bearing reminder that this is a regulated timeline. A static "EU 178/2002 — 4 h response" badge would not communicate urgency at minute 230 the way `01:30 restante` does.
- **`--destructive` is the CTA bg, not the trace tree.** The trace tree is data, not danger. Painting it red would make every recall surface look like every other recall surface and would dull the cue when the CTA actually fires. The destructive paprika belongs on the action, not on the read surface.
- **Forward-trace is the default; reverse-trace is one tap away.** The 2 AM case is "what got served from this lot?". The reverse case ("which supplier sent us this contaminated batch?") is the morning's question, secondary on this surface. A secondary chip below the tree toggles to reverse mode; the geometry of the tree inverts (FR16). Same component, two modes — not two separate surfaces.
- **Lot codes in Fraunces serif.** The one-serif-wink rule (DESIGN.md §3) applied to the lot identifier so the eye locks onto it as the canonical reference. Everything else stays system sans.
- **Phone-grade portrait layout is the canonical mock.** The 2 AM operator is on a phone, not at a desk. The mock optimises for portrait at 390 × 844 (iPhone reference). Landscape and laptop adaptations are documented in implementation notes but do not change the layout primitives.
- **No "edit dossier" affordance pre-dispatch.** Pre-formatted is the point — every field in the dossier is sourced from the recall query and audit_log. Letting the operator edit at 02:00 invites mistakes that the regulator will read. Post-dispatch the dossier is amendable via [J7](j7.md) (an addendum flow), with the original immutably preserved.
- **The audit_log envelope for `recall.dispatch-86-flag` carries the full incident dossier as `payload_after`.** Per audit-log canonical (ADR-025) and M3 audit_log extension (ADR-031): the dispatch row is the canonical chronological evidence the regulator inspects. The dossier PDF is a derived artefact; the truth is in audit_log.

## Notes for implementation

- **First-paint < 800 ms on 4G**. The crisis surface MUST NOT wait on a slow query. The search candidates load on user input (not pre-fetched); the empty state is the initial render. The four-hour clock fires from server time on first render to avoid clock-skew anxiety.
- **`RecallTraceTree` uses the SQL recursive CTE described in ADR-028.** The component receives a flattened tree payload from the recall API; rendering is a single pass. Indexes per ADR-031 (compound `(organization_id, lot_code)` + traversal indexes) make the query p95 < 500 ms at 100k events (NFR-PERF-1).
- **`recall.dispatch-86-flag` MCP capability** is the same surface Hermes hits from WhatsApp / Telegram (FR42 — same MCP, multiple agent surfaces). Iker's web tap and a notional voice-driven "Hermes, corta el servicio en ese lote" route to the identical handler. The web mock here is one rendering of the same capability.
- **`prefers-reduced-motion` disables every transition.** Even at 150 ms ease-out-quart, a transition on the crisis surface can read as a hesitation. Reduced-motion users get instant state swaps. The countdown clock continues to tick (it is not a motion effect; it is the data).
- **`@media (prefers-color-scheme: dark)` is honoured.** At 02:00 a bright cream canvas is hostile. Dark mode rebalances `--bg` to `oklch(20% 0.012 70)` (warm charcoal) with `--ink` flipped to `oklch(94% 0.010 60)` (warm cream); the paprika `--destructive` stays paprika (it is the signal, not the chrome). Token definitions in DESIGN.md §2 with dark-mode overrides.
- **The crisis surface is exempt from the standard top-nav.** Routes that begin `/recall/*` mount on a dedicated layout shell (`apps/web/src/layouts/CrisisLayout.tsx`) — no header, no sidebar, no global notifications. Other M3 surfaces (J7, J8, J9, J10, J11, J12) use the standard layout.
- **Accessibility.** All interactive elements are reachable via keyboard tab order (search field → first candidate row → trace nodes → CTA). The `RecallTraceTree` uses `role="tree"` with `aria-level` per depth. The CTA is `<button>`, never a styled `<a>`. WCAG-AA verified in the mock head comment.
