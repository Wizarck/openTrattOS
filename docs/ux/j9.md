# J9 — Owner generates the APPCC audit export bundle (compliance path)

The export trigger surface. Marta arrives quarterly; the call to Iker is "I need the last 90 days HACCP + lot + corrective actions in the standard format, in Basque, by Friday". Iker comes here, picks the range + locale + scope, generates the bundle, and either downloads it locally or emails it directly to Marta's pre-configured contact. The visual surface is [mock-j9-appcc-export.html](variants/mock-j9-appcc-export.html); this doc anchors that mock to the export contract.

## Goal

In ≤90 seconds Iker (i) picks the date range + autonomous-community locale + scope, (ii) triggers bundle generation, (iii) watches a progress strip, and (iv) downloads PDF + CSV companion (and optionally emails them). The bundle Marta receives is the **unedited** audit_log as chapter 0, plus structured derivative chapters on top — never a curated executive summary.

## Trigger

Iker opens `/compliance/export` from the top nav. Alternatively, the dashboard auto-opens with a pre-filled `last 90 days` range when an inspector pre-read request arrives by email (deep-link to this surface with query params).

## Walkthrough

The numbers below correspond to the regions of [mock-j9-appcc-export.html](variants/mock-j9-appcc-export.html).

1. **Header + transparency banner.** Eyebrow: `Exportación APPCC · expediente para autoridad sanitaria`. Headline: `Generar bundle de auditoría`. Below the headline, a `--mute` paragraph that surfaces the contract verbatim: *"El expediente contiene el audit_log sin editar como capítulo 0; el resto son vistas estructuradas sobre ese mismo registro. No producimos resumen ejecutivo."* This text is load-bearing — it tells Marta (the inspector reading the cover page) and tells Iker (the operator generating it) what they will receive.
2. **Date range picker.** Two adjacent date inputs, `Desde` and `Hasta`. Default = previous 90 days (quarterly cadence). Quick chips below: `Últimos 30d`, `Últimos 90d` (selected), `Año natural en curso`, `Trimestre cerrado`. Custom dates allowed. (FR21 — configurable range.)
3. **Locale picker.** Four chips, single-select: `Castellano (es-ES)` (default), `Català (ca-ES)`, `Euskara (eu-ES)`, `Galego (gl-ES)`. The choice swaps every template string + the EU 1169 Annex II allergen vocabulary table (ADR-035). The chip group shows a mute footer line: `La localización determina el idioma de los encabezados, etiquetas, y vocabulario de alérgenos.` (FR23, FR24.)
4. **Scope checkboxes.** Five checkbox rows: `HACCP records (CCP readings + correctivas)` (checked default), `Lot lifecycle (recepción → consumo)` (checked), `Procurement (PO + GR + reconciliación)`, `Photo-ingestion provenance` (unchecked default), `AI observability footprint`. Each row carries a one-line `--mute` description. Scope determines which chapters are added to the bundle on top of chapter 0 (raw audit_log). (FR22 — configurable scope.)
5. **Recipient picker (optional).** A collapsed strip: `Enviar también por email →`. Expanded, it shows pre-configured inspector + insurer contacts as checkboxes; operator can add ad-hoc recipients. Default = unchecked — the operator may want to download first and email after.
6. **Generate button.** Primary `--accent` CTA: `Generar bundle`. Sits sticky at the right of the action row. Tap → progress strip mounts below.
7. **Progress strip.** Replaces the Generate button with a stepped progress display: `Indexando audit_log… ▸ Componiendo capítulo 0 ▸ Renderizando vistas derivativas ▸ Sellando hash de bundle ▸ Listo`. Each step lights `--success` as it completes. Bundle size + page count update live (`~48 páginas · ~2.3 MB`). On completion the strip morphs to a download row.
8. **Download row.** Two ghost buttons: `Descargar PDF (es-ES) · 2.3 MB · 48 páginas` and `Descargar CSV companion · 412 KB`. A mute eyebrow above: `Bundle generado a las 14:32 CEST · firma SHA-256 a9f3…b274 · audit_log entry AL-2026-189554`. If email was selected, a third row reads `Enviado a 2 destinatarios · ✓ entregado`.
9. **Past bundles archive (below the trigger area).** A flat table of the last 10 bundles: date generated, range, locale, scope summary, who generated, download link. Persists indefinitely (ADR-029 retention archival applies — bundles older than the retention window live in cold storage with a `restaurar →` link). Marta can refer Iker to "the bundle you sent me in March" and Iker finds it in 1 click.

## Capabilities satisfied

FR21 (configurable date range), FR22 (configurable scope), FR23 (multilingual templates es/ca/eu/gl per ADR-035), FR24 (EU 1169 Annex II allergen vocabulary per locale), FR25 (raw audit_log chapter 0 + structured derivative chapters), FR19 (email dispatch via ADR-039), NFR-OBS-1 (warning span on locale fallback). Cross-reference [PRD-M3 §FR Catalogue Regulatory Compliance & Export](../_bmad-output/planning-artifacts/prd-m3.md).

## Edge cases

- **Range > 365 days.** The form accepts it; the generator warns `Rango > 1 año puede tardar varios minutos` in a mute footer line. No hard block. Bundle generation runs as a background job; the operator may close the tab and return later (the bundle appears in the archive when done). Email dispatches automatically if recipients were configured.
- **Locale = eu-ES but template not yet seeded for some keys.** Per ADR-035 fallback chain: missing key → `es-ES` default → emit warning span (NFR-OBS-1). The bundle still generates; a mute eyebrow on the download row reads `Algunas etiquetas no disponibles en euskara · vea log de localización →`.
- **Scope = `Photo-ingestion provenance` but zero photos in range.** That chapter renders as a single line: `Sin fotos de aprovisionamiento en este rango.`. The chapter is included for completeness (so the operator can prove nothing was hidden); empty is honest.
- **Bundle generation fails mid-render.** Progress strip rolls back the last step's success state to `--destructive`. The operator gets `Generación falló · ver log →`. The partial bundle is discarded; no half-bundle is downloadable. A retry button mounts inline.
- **Email dispatch fails after bundle generates.** The bundle is still downloadable; only the email side failed. Per ADR-039 the email service retries 3× with exponential backoff. Status surfaces in the download row's email line.
- **Manager (not Owner) attempts to generate.** Per RBAC (ADR-006 inheritance): Manager has scope-restricted export — they can only generate for their assigned locations. The scope checkboxes implicitly filter; a mute eyebrow at the top reads `Tu alcance: 1 de 3 locales`. If the inspector needs the org-wide bundle, Manager escalates to Owner.
- **Two operators trigger simultaneously.** Each gets its own bundle (independent SHA-256). The archive shows both. No serialisation, no queue.

## Decisions specific to J9

- **Transparency banner is verbatim and load-bearing.** Marta reads the cover page first; the contract — "no executive summary; chapter 0 is raw" — must appear there and on this surface. A polished marketing description ("Generate a beautiful compliance report") would be wrong: the inspector wants to be told what they will not get, so they know to do their own analysis. (PRD-M3 §Journey 5 trust principle, locked.)
- **Default range = last 90 days.** Quarterly inspection is the recurring case. Year-natural and quarter-closed chips exist but are not the default — defaults serve the recurring case. (DESIGN.md §1 principle 8.)
- **Locale chips, not a dropdown.** Four options + visual permanence supports the autonomous-community context (the operator may need to glance at the locale they picked while configuring the rest). A dropdown hides the choice the moment it closes; the chip stays visible.
- **Scope is checkboxes, not a multi-select dropdown.** Same reason — every dimension of the export visible at-glance. Checkboxes also support partial selections naturally (the dropdown pattern would force "all or pick subset" mental work).
- **Email is opt-in, not the default action.** Many bundles are downloaded for the operator to review first before forwarding. Making email the default would lead to accidental sends. (DESIGN.md §1 principle 8 — boring before clever.)
- **Past bundles persist in an archive table on this surface.** Inspectors refer to past bundles ("the one you sent in March"). Surfacing the archive on the same surface where new bundles are generated keeps that reference cheap. A separate "/compliance/archive" route would be a click the operator does not need.
- **Bundle SHA-256 surfaces inline on the download row.** Marta can verify the hash matches her downloaded file — chain-of-custody at her fingertips. The hash also feeds the audit_log envelope so the chain is forensically intact.
- **No "preview the PDF before generating" affordance.** Generation is the action. Pre-generation preview would imply the export could be edited before dispatch — exactly what the trust principle forbids. Operators preview AFTER generation via the download.
- **Locale switch does not regenerate already-generated bundles.** Each bundle is locked to the locale active at generation time. The archive preserves the historical locale per row. If the inspector needs a Basque version of a bundle that was generated in Spanish, the operator generates a new bundle for the same range in eu-ES.

## Notes for implementation

- The bundle generator is a NestJS job that runs synchronously for ranges ≤90d, async for > 90d. Async progress streams via Server-Sent Events to the progress strip.
- Chapter 0 (raw audit_log) is rendered by streaming SELECT * from audit_log → CSV first, then PDF wrapping. Memory-efficient for large date ranges.
- Locale fallback chain implemented per ADR-035: per-key lookup → `es-ES` default → `WARN` span emit. The warning is in the audit_log of the bundle generation, NOT in the bundle itself (would leak operator-side telemetry into the regulator-facing artefact).
- The CSV companion is canonical UTF-8 with BOM; opens correctly in Excel ES locale.
- Bundle SHA-256 computed over the concatenated `(pdf_bytes || csv_bytes)`. Stored on the `audit_log_envelope.payload_after.bundle_sha256`. The archive table reads from this column for the hash display.
- Bundle archive table reads from the same envelope chain; no separate `bundles` table. (Single source of truth — ADR-025.)
- Email dispatch uses `EmailDispatchService` per ADR-039. Attachments inline; subject + body templated per locale.
- First-paint < 1 s. The locale picker and scope checkboxes preload their state from the operator's last successful generation (a small ergonomic).
- Accessibility: the progress strip uses `role="status"` with `aria-live="polite"` so screen readers announce each step. Date inputs use native browser pickers + ARIA labels.
